# Story 1.1 - React + Vite 앱 골격과 단일 페이지 레이아웃 만들기

## 구현 목표

입문자 강의의 첫 구현 사이클에서는 앱 기능을 모두 만들지 않는다. 30분 안에 끝낼 수 있도록, React + Vite 앱을 만들고 브라우저에서 한 화면짜리 정적 레이아웃이 보이는 상태까지 구현한다.

이 story가 끝나면 사용자는 아직 입력을 저장하거나 계산하지는 못하지만, 앞으로 채워 넣을 방문 정보, 굿즈 예산, 체크리스트, 생존 플랜 카드, 최근 플랜 저장 영역을 한 화면에서 볼 수 있다.

## 포함 범위

- React + Vite 단일 페이지 앱 scaffold 생성
- `src/App.jsx`에 앱 제목과 주요 영역 배치
- `src/styles.css` 또는 기존 CSS 방식으로 기본 레이아웃 작성
- 모바일 폭에서 세로로 쌓이는 레이아웃 작성
- 제외 기능이 화면에 들어오지 않았는지 확인

## 제외 범위

- 방문 날짜/시간대 입력 동작
- 예상 대기 시간 메시지 계산
- 굿즈 항목 추가/삭제
- 체크리스트 체크/해제
- 생존 플랜 카드의 실제 계산 로직
- localStorage 저장/복원
- 로그인, 결제, 지도, 실시간 재고, SNS 공유, 서버 저장

## 구현 메모

- 상태 관리는 아직 추가하지 않아도 된다.
- 새 상태 관리 라이브러리, 라우터, 외부 API 의존성을 추가하지 않는다.
- 화면 문구는 초보자가 바로 이해할 수 있게 짧게 쓴다.
- 생존 플랜 카드 영역은 빈 상태 안내 또는 기본 문구만 표시한다.
- 체크리스트 영역에는 기본 항목 이름만 정적으로 보여줘도 된다.
- 기본 체크리스트 문구는 `보조 배터리`, `물`, `입장 정보 확인`, `카드/현금 준비 확인`, `굿즈 보관 가방`을 사용한다.

## Acceptance Criteria

1. `npm install` 후 `npm run dev`로 앱이 실행된다.
2. 브라우저 첫 화면에 앱 제목이 보인다.
3. 한 페이지 안에 다음 5개 영역이 보인다: 방문 정보, 굿즈 우선순위와 예산, 현장 체크리스트, 오늘의 생존 플랜 카드, 최근 플랜 저장.
4. 로그인, 결제, 지도, 실시간 재고, SNS 공유, 서버 저장을 암시하는 버튼이나 링크가 보이지 않는다.
5. 브라우저 폭 375px에서 주요 영역이 세로로 쌓이고 가로 스크롤이 생기지 않는다.
6. 개발자 도구 Console에 초기 렌더링 오류가 없다.

## 검증 방법

```bash
npm install
npm run dev
```

- 브라우저에서 로컬 개발 서버를 연다.
- 첫 화면에 5개 영역이 모두 있는지 확인한다.
- DevTools 또는 브라우저 반응형 모드에서 폭을 375px로 줄이고 가로 스크롤이 없는지 확인한다.
- Console에 빨간 오류가 없는지 확인한다.
