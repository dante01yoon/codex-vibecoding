# AGENTS.md

## Project

이 프로젝트는 React로 만드는 "오늘의 코딩 퀘스트" 미니 앱입니다.

사용자는 버튼을 눌러 오늘의 코딩 퀘스트 카드를 랜덤으로 뽑고, 카드 등장/퇴장 애니메이션과 버튼 눌림 효과를 확인할 수 있습니다. 1차 구현은 작고 명확하게 동작하는 버전을 목표로 합니다.

## Working Rules

- 작고 집중된 변경을 우선합니다.
- 구현 전에 확인하거나 수정할 파일을 먼저 설명합니다.
- 방금 확인한 Motion for React 문서 내용을 기준으로 구현합니다.
- 첫 구현은 단순하지만 실제로 동작해야 합니다.
- 작업 후에는 변경 파일, 문서 참고 내용, 확인 방법, 남은 위험을 요약합니다.

## Product Scope

- 오늘의 코딩 퀘스트 카드를 표시합니다.
- 랜덤 뽑기 버튼으로 다른 퀘스트를 보여줍니다.
- 카드가 나타날 때 등장 애니메이션을 적용합니다.
- 카드가 교체되거나 사라질 때 퇴장 애니메이션을 적용합니다.
- 버튼에는 눌림 효과를 적용합니다.
- 퀘스트 완료/미완료 상태를 토글할 수 있습니다.

## Motion for React

- 추가 가능한 애니메이션 라이브러리는 `motion`입니다.
- 설치 명령은 `npm install motion`입니다.
- React import 경로는 `motion/react`입니다.
- 기본 import 예시는 `import { motion, AnimatePresence } from "motion/react"`입니다.
- 일반 HTML 요소는 `motion.div`, `motion.button` 같은 motion component로 바꿔 사용합니다.
- `initial`은 처음 나타나기 전 상태를 정의합니다.
- `animate`는 화면에 보일 때 도착할 상태를 정의합니다.
- `exit`는 요소가 제거될 때의 퇴장 상태를 정의합니다.
- `whileTap`은 버튼을 누르는 동안의 인터랙션 효과를 정의합니다.
- `AnimatePresence`는 조건부 렌더링되는 카드의 `exit` 애니메이션을 실행하기 위해 사용합니다.

## Do Not

- `framer-motion`을 설치하거나 import하지 않습니다.
- `motion` 외의 새 라이브러리를 추가하지 않습니다.
- 드래그 제스처를 구현하지 않습니다.
- 스크롤 기반 애니메이션을 구현하지 않습니다.
- 복잡한 variants 구조나 layout 애니메이션을 1차 구현에 넣지 않습니다.
- 데이터 저장, 검색, 필터, 카테고리 관리, 서버/API 연동을 추가하지 않습니다.
- 작은 요청을 처리하기 위해 프로젝트 전체를 다시 작성하지 않습니다.

## Verification

- 앱이 정상 실행되는지 확인합니다.
- 랜덤 뽑기 버튼을 눌렀을 때 카드 내용이 바뀌는지 확인합니다.
- 새 카드가 `initial`에서 `animate` 상태로 자연스럽게 등장하는지 확인합니다.
- 카드 교체 시 `AnimatePresence`와 `exit` 애니메이션이 동작하는지 확인합니다.
- 버튼을 누를 때 `whileTap` 효과가 보이는지 확인합니다.
- 완료 상태 토글이 화면에 즉시 반영되는지 확인합니다.
- `framer-motion`이 설치되거나 import되지 않았는지 확인합니다.
