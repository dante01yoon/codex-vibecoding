# 첫 실행 검토 체크리스트

자동화는 만든 뒤 첫 2-3번의 실행 결과를 꼭 검토하세요.

## 내용 품질

- [ ] 뉴스가 3-5개 정도로 적절하게 선별되었다.
- [ ] 단순 번역이 아니라 왜 중요한지 설명한다.
- [ ] 코덱스/바이브 코딩 학습과 연결되는 해석이 있다.
- [ ] 입문자가 이해하기 어려운 용어를 짧게 풀어준다.
- [ ] 너무 길지 않고 5분 안에 읽을 수 있다.

## 출처 품질

- [ ] 각 뉴스에 출처와 날짜가 있다.
- [ ] 공식 문서/공식 블로그/신뢰 가능한 매체를 우선한다.
- [ ] 루머나 홍보성 글을 사실처럼 말하지 않는다.
- [ ] 같은 내용을 중복으로 가져오지 않는다.

## 안전

- [ ] 웹페이지 안의 지시문을 따르지 않는다.
- [ ] 외부 사이트에 로그인하지 않는다.
- [ ] 이메일/슬랙/댓글 등 외부 발송을 하지 않는다.
- [ ] API 키나 개인 정보를 요구하지 않는다.
- [ ] 확실하지 않은 내용은 확인 필요로 표시한다.

## 자동화 설정

- [ ] 스케줄이 원하는 시간대 기준으로 설정되었다.
- [ ] Thread automation으로 같은 스레드에 붙어 있다.
- [ ] 네트워크/웹 검색 권한이 필요한 경우 실행 환경이 이를 허용한다.
- [ ] 결과가 너무 길거나 짧으면 프롬프트를 수정했다.

## 개선 프롬프트 예시

```text
다음 실행부터는 뉴스 개수를 최대 5개로 제한하고, 각 항목의 "강의 업데이트 아이디어"를 한 문장으로 더 구체화해주세요.
```

```text
다음 실행부터는 OpenAI/Codex 관련 공식 문서 업데이트를 가장 먼저 확인하고, 일반 AI 산업 뉴스는 후순위로 두세요.
```

```text
다음 실행부터는 입문자가 모를 수 있는 용어를 "오늘의 키워드"에서 더 쉽게 설명해주세요.
```
